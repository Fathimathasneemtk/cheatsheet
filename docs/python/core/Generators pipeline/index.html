Generators pipeline

1- Generators​

  • Generator is special type of function that returns an iterator object.
  • Insted of using return keyword to send single value, generator function uses yeild to produce series of values overtime.
  • The Generator function pauses its execution after yield, maintaining (retaining or freezes) its state between iterations.
  • Each call to next() on generator object , resumes its execution right afetr the yield statement.
  • Example 
    def func(max):
      cnt=1
      while cnt <=max:
          yield cnt
          cnt+=1

    ctr=func(5)
    for n in ctr:
      print(n)

    output 
    1
    2
    3
    4
    5

2 - Need of Generators​

Memory efficeint​

Handle large or infinite data without loading everything into memory.

NO list overhead​

Yield items one by one at a time ,avoiding full list creation.

Lazy Evaluation​

Lazy evaluation measns "Dont do the work, dont use the memory until the code absolutely asks for it". So only while calling next() , then next iteration happens.

Support infinfte sequences​

Ideal for generating unbounded data like Fibonacci series.

Pipeline processing​

Chaining generators (built by passing one generator into another.) , to process data in stages efficeintly.

  • The scinario cleaning a list of text

    • cleain up extra spaces

    • Capitalize the names

    • Keep only the names start with the letter 'A'

      Here is how the generator chains together.

      # Our raw data source
      raw_names = ["  alice ", "  bob  ", " charlie ", " alexandRA  "]

      # Stage 1: Strips white spaces
      strip_stage = (name.strip() for name in raw_names)

      # Stage 2: Capitalizes the words
      caps_stage = (name.capitalize() for name in strip_stage)

      # Stage 3: Filters only 'A' names
      final_pipeline = (name for name in caps_stage if name.startswith("A"))

      # --- AT THIS POINT: Zero work has been done. ---
      # The pipeline is built, but nobody has asked for data yet.

      print("Starting the loop...")
      for name in final_pipeline:
          print(f"Result: {name}")

      The data moves as "Pull effect". When the for name in fina_pipeline loop runs, it pulls from the end of the line.Here is the exact chain reaction.

      [Loop] says: "Hey Final Stage, give me a name!"
      ↓
      [Final Stage (Filter 'A')] says: "I don't have one yet. Hey Caps Stage, give me a capitalized name!"
      ↓
      [Caps Stage] says: "I don't have one either. Hey Strip Stage, strip something for me!"
      ↓
      [Strip Stage] grabs "  alice " from the raw list, strips it to "alice", and hands it to Caps Stage.
      ↓
      [Caps Stage] takes "alice", turns it into "Alice", and hands it to the Final Stage.
      ↓
      [Final Stage] checks "Alice". It starts with 'A'! It hands it to the Loop.
      ↓
      [Loop] prints: "Result: Alice"

      Why this is highly efficeint​

      If you did this with regular python list, Python would:

      • Loop through all names to strip and save a new list to memory.
      • Loop through all stripped names to capitalizes them and save another new list to memory
      • Loop through all the capitalized names to filter them and save a third list to memory.

    • With Generator Pipeline, there are no intermediate lists are created in computer's RAM.It is a single continues stream where one item goes all the way through the pipeline before next item even enters.

3- Yield Vs Return​

  • Yeild :​

    It is used in generator function to provide a sequence of values over time.When yield is executed it pauses the function, returns the current value and retains the state of the function.This allows the functions to continues from same point when called again through next(), making it ideal for generating large or complex sequences efficeinlty.

  • Return:​

    It is used to exit from the function and return a final value.Once return is executed , function is terminated immediately and no state is retained.This is suitabel for cases where single value is neede from a function.

  • Generator expression are concise(short) way to create generators.
  • They are similar to list comprehensation but use parentheses instead of square brackets and more memory efficeint.
  • Example

    for in sq:

5- Shortcut tools​

  1. yield from​

    yield from is a shortcut used inside the generator to yield all values from another iterable (list,tuple, or another generrator) without using the for loop.
    • old way
    def combine_lists_old():
    list_a = [1, 2]
    list_b = [3, 4]

    # We have to loop through list_a manually
    for num in list_a:
        yield num
        
    # We have to loop through list_b manually
    for num in list_b:
        yield num

    # Test it
    for x in combine_lists_old():
        print(x)  # Outputs: 1, 2, 3, 4
    • The cleaner way
    def combine_lists_new():
    list_a = [1, 2]
    list_b = [3, 4]

    # Shortcut! "Yield everything inside list_a, then move on"
    yield from list_a
    yield from list_b

    # Test it
    for x in combine_lists_new():
       print(x)  # Outputs: 1, 2, 3, 4
    • WHY USEFULL
      • Saves line of code
      • Perfect for flatteinig the data
        • If you have nested structures (like a list of lists) or chained generators in a pipeline, yield from lets you seamlessly pass data up the chain without nesting multiple for loops

  2. send() Method​

  • Normally generator only give data using ```yield`` .
  • The send() method make them 2 way street
  • It allows to inject a data value back inside the generator exactly where it last paused
  • The value we send actually becomes the result of the yield statement inside the function.

def dynamic_counter():
  count = 0
  while True:
      # 1. Python pauses right here when it hits 'yield count' and outputs the value.
      # 2. When you use .send(), the received value drops into the 'received' variable.
      received= yeild count
      
      # If the user sent something, update our count!
      if received is not None:
          count = received
      else:
          count += 1

 # --- Using the generator ---
 gen = dynamic_counter()

 # STEP 1: The Prime Rule (You must always start/prime a generator)
 # You can't send data immediately because the generator hasn't paused at a 'yield' yet.
 print(next(gen))  # Outputs: 0 (Pauses at the first yield)

 # STEP 2: Let it run normally
 print(next(gen))  # Outputs: 1 (received is None, so it increments by 1)
 print(next(gen))  # Outputs: 2


 # STEP 3: Use .send() to inject a value!
 # This changes 'received' to 100 inside the function
 print(gen.send(100))  # Outputs: 100 

 # STEP 4: Continue normally from the new starting point
 print(next(gen))  # Outputs: 101

The rules of .send()​

  1. You must "prime"(start) it first​

  • When we create a generator it is sitting at the top level of the function.It has not hit a yield statement yet.
  • You must call a next(gen) or gen.send(None) before doing the .send(value) .Otherwise if you first call gen.send(100) will raise TypeError.
  • The yield statement should come first for catching it ,before we sending. If there is no yeild statement then it will crashes with TypeError.

  1. next(gen) is just a short call​

    Calling next(gen) is same as calling gen.send(None)

Application:​

In advanced programming, .send() is the foundational building block for coroutines and light weight cooperative multitasking. It allows different parts of the program to pause, pass data back and forth, coordinate tasks without running multiple heavy threads at the same time.

6- Advantages of Generators​

  • Useful for procesing large files like logs, because generator handle it in smaller parts ,only holds one logs in memory at any given second, saving memory and we dont wants to load the entire file at once.
  • The iterator can do similar tasks, Generators are quicker bcz of we dont wants define next and iter methods manually.

7- Generator VS Iterators​

A. The Core Difference​

  • Iterator : A generic concept (a python object) used to loop over the sequence.To make custom iterator , we have to build a class with two methods __iter__() and __next__().Also we need to manage the internal state.
  • Generaot : A simple way to create an iterator.Instead of writing the whole class , just write a regular function with yield keyword instead of return .Python automatically handles __iter__() and __next__() mechanism for you behind the scenes.

B. Quick Comparison​

FeatureIteratorGenerator
ImplementationClass based (requires __next__() and __iter__())Function based (uses keyword yield)
State ManagementManual (You to tarck variables using self)Automatic (Python saves the local variable state)
ComplexityMore boilerplate code: HarderClean,readable and highly Pythonic
Memory efficeincyHigh(Compues values on the fly(only when actually needed))High(Computes values on the fly(only when actually needed))

NOTE : Iterator is different from Iterable

  • Iterable(The Book) : The actual data collection(list, tuple or set).It contains all data ,it holds millions of records in the Computer memory, It is not memory efficeint.
  • Iterator(The Bookmark) : This is a separate tiny object whose only job to point to one specific spot in a collection and know how to move to the next one.So Memory efficeint.

C. Side by Side Code Example​

a. The iterator way(Class)​

We ahve to track the state (self.current) yourself and manually raise StopIteration excpetion when done.


class CountUp:
  def __init__(self,max_num):
    self.max_num=max_num
    self.current=1

  def __iter__(self):
    return self

  def __next__():
    if self.current <=self.max_num:
      result=self.current
      self.current+=1
      return result
    else:
      raise StopIteration

b. The Generator way (Function)​

Everytime Python hits a yield keyword, it pauses the function execution, saves its state ,and spit out the value.When called again, it resumes right where it left off.


def count_up(max_num):
  current=1
  while current <=max_num:
    yield current
    current+=1

-> How use you Both

# They are used exactly the same way!
for num in count_up(3):
    print(num)  # Outputs: 1, 2, 3
