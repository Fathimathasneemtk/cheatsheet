OOPs

  • OOP is a way of organizing code that uses classes and objects to represent real world entities and their behavior.
  • Object has attribute thing that has specific data and can perform certain actions using methods.
  • Organizes code into object and classes
  • Supports encapsulation which group data and methods together.
  • Enables inheritance for resusability and heirarchy
  • Allows Polymorphism for flexible method implementation
  • Improves modularity, scalabilty and maintainabilty.

Class​

Class is the collection of objects. Class defines a set of attributes and methods which the created objects have.

  • Classes are created by keyword class
  • Attributes are the variable belongs to a class.
  • Attributes are always public and can be accessed by using dot(.) operator.
  • class attribute is shared by all instance of the class.
  • __init__() is a constructor method which runs automatically when a new instance is created.It is used to initialize object data.
  • self represents current object allowing each instance to store and access its own data.
  • instance attribute is unique to each objects of the class.

class Dog:
    species = 'Canine' # class attribute

    def __init__(self,name,age):
        self.name=name # instance attribute
        self.age=age # instance attribute

Objects​

Object is the instance of the class , each object holds its own data.he objects consists of

  • State : It is represented by the attributes and refelects the properties of an object.
  • Behavior : It is represented by the methods of an object, and reflects the response of an object to other objects.
  • Identity : It gives a unique name to an object and enables one object to interact with other objects.
# 1. We create the objects
my_phone = Smartphone("iPhone 15")
abhishek = Person("Abhishek")

# 2. The interaction happens
abhishek.call_friend(my_phone)

Creating of an object​

sFour Pillars of OOP

1. Inheritance​

Inheritance allows a class to acquire properties and methods of another class.It promotes code reuse and hierarchical classification.

TYPES OF DIFFERENT INHERITANCE

A. Single Inheritance​

A single child inherits from a parent parent class.

class Parent:
    def feature_one(self):
        return  "Feature 1 from Parent"

class Child(Parent):
    def feature_two(self):
        return "Feature 2 from Child"

obj=Child()
print(obj.feature_one()) # Feature 1 from Parent ----- Inherited

B. Multiple Inheritance​

This is where a child class inherits from more than one Parent class.

NOTE:This is superpower unique to C++, Python .Java doesn't allow this because it can cause problem known as Diamond Problem(Child class wouldn't know which parent's method to run if they both had same name).Python solves this using an order system called MRO(Method Resolution Order).

class Father:
    def skills(self):
        return "Coding"

class Mother:
    def hobbies(self):
        return "Painting"

class Child(Father, Mother):  # Inherits from BOTH
    pass

kid = Child()
print(kid.skills())   # From Father: Coding
print(kid.hobbies())  # From Mother: Painting

C. Multilevel Inheritance​

A child class inherits from a parent class , which in turn inherits form a grandparent class.

class Grandparent:
    def land(self):
        return "Owned 100 acres"

class Parent(Grandparent):
    def house(self):
        return "Built a house on the land"

class Child(Parent):
    def car(self):
        return "Bought a car to park at the house"

kid = Child()
print(kid.land())   # Trickled down from Grandparent
print(kid.house())  # Trickled down from Parent


D. Hierarchical Inheritance​

One parent class serves as the base for multiple child classes.The child shares the parent features, but they dont share feartures each other.

class Vehicle:
    def info(self):
        return "This is a transport vehicle."

class Car(Vehicle):
    def speed(self):
        return "Top speed 120 mph"

class Truck(Vehicle):
    def cargo(self):
        return "Can carry 5 tons"

# Both Car and Truck get 'info()', but Car doesn't get 'cargo()'

E. Hybrid Inheritance​

Hybrid inheritnce is a combinatin of two or more inheritance listed above. eg: A mix of Multiple and Multilevel inheritance.

class School:
    def school_name(self): return "High School"

class Faculty(School):  # Hierarchical/Multilevel branch
    def work(self): return "Teaching"

class Sports:           # Independent base
    def activity(self): return "Basketball"

class Student(Faculty, Sports): # Multiple Inheritance branch
    pass

2. Polymorphism​

Same operation but different behavior.

  • It allows different classes to have functions or methods with the same name to work differently depending on the type of object they acting upon.
  • Python handles this smoothly because of python in dynamically typed.
class Duck:
    def sound(self):
        return "Quack"

class Dog:
    def sound(self):
        return "Woof!"

def make_it_speak(animal_obj):
    animal_obj.sound()


make_it_speak(Dog()) # Woof!
make_it_speak(Duck()) # Quak!

In Pyhton Polymophism is flexible ,usually shows up two main inheritance. With Inheritance and Without Inheritance.

A. Polymorphism with inheritance​

This is a class-object oriented concept.Multiple child class inherits from a parent class, but they override a specific method to change its behaviour.


class Document:
    def __init__(self,name):
        self.name=name
    
    def show(self):
        pass

class WordDocument(Document):
    def show(self):
        return f"Opening {self.name} in MIcrosoft Word Document."

class PDFDocument(Document):
    def show(self):
        return f"Opening {self.name} as PDF layout."

def open_file(obj):
    print(obj.show())

open_file(WordDocument("Resume.docx"))
open_file(PDFDocument("Ebook.pdf"))

  • Python is dynamically typed,Which gives a unique flaovour of inheritance called ** Duck Typing**.The phrase comes from saying :"If it walks like a duck and quacks like duck , its a duck".

  • Python doesn't care about the type of the class or Family tree it only care abouts the object has the method it wants to call.


class Duck:

make_sound(Duck("maju"))

So in Java was this it wont work for otherthan what type it mentioned in the function.Suppose for this make_sound only accepts object from Animal class , then it wont accept CarHorn class object.

C. Polymophism in Built-in Python Functions​

We using polymophism in everyday, without realizing it. Built-in functions like len() change their behavior depending on what you give them.


# len() counts characters in a string
print(len("Python"))  # Output: 6

# len() counts items in a list
print(len([1, 2, 3]))  # Output: 3

# len() counts key-value pairs in a dictionary
print(len({"a": 1, "b": 2}))  # Output: 2

Method Overloading Vs Method Overriding​

A. Method Overriding​

Redefining a parent class's method inside a child class.

B. Method Overloading​

Having multiple functions in the same class with the exact same name but different input arguments.eg: abc(x,y,z) and abc(x,y).

NOTE: Python doesn't support native method overloading.If we write 2 functions with the same name , the second one simply overrides the first one.To mimic overloading we use default args (like def add(x,y,z=9) or variable arguments *args).

NAtive Method Overloading -Python Not Supports​

Python reads code from top to bottom. When it encounters the second function, it simply points the function name to the new definition, completely erasing the first one.

Why can't Python do this?

  • Python is a dynamically typed language and functions don't enforce input data types. Furthermore, a Python class is essentially a dictionary under the hood. When you define a method, Python stores it as a key-value pair: {"add": function_object}.

  • If you define a second method named add, Python simply updates that dictionary key to point to the new function, completely erasing the old one. Java, on the other hand, keeps a structured list of distinct method signatures.

class Calculator:
    # First definition (takes 2 arguments)
    def add(self, x, y):
        print("Running the 2-argument version...")
        return x + y

    # Second definition (takes 3 arguments)
    # Python looks at this and says: "Oh, 'add' means THIS now!"
    def add(self, x, y, z):
        print("Running the 3-argument version...")
        return x + y + z

calc = Calculator()

# 1. Testing the 3-argument version (Works perfectly)
print(calc.add(5, 10, 20))
# Output:
# Running the 3-argument version...
# 35

# 2. Testing the 2-argument version (CRASHES!)
print(calc.add(5, 10))

Method Overloading in JAVA - supports​

In Java, this code works perfectly. Java supports native method overloading because it uniquely identifies a method by its Signature.

A method signature in Java isn't just the name of the function; it is a combination of:

  • The method name.

  • The number of parameters.

  • The data types of those parameters.


class Calculator {
    // Method 1: Signature is "add(int, int)"
    public int add(int x, int y) {
        System.out.println("Running 2-argument version...");
        return x + y;
    }

    // Method 2: Signature is "add(int, int, int)"
    // Java sees this and says: "This is a brand new method!"
    public int add(int x, int y, int z) {
        System.out.println("Running 3-argument version...");
        return x + y + z;
    }

    public static void main(String[] args) {
        Calculator calc = new Calculator();

        // Java checks the inputs: 2 integers -> runs Method 1
        System.out.println(calc.add(5, 10));      
        // Output: 
        // Running 2-argument version...
        // 15

        // Java checks the inputs: 3 integers -> runs Method 2
        System.out.println(calc.add(5, 10, 20));  
        // Output: 
        // Running 3-argument version...
        // 35
    }
}

a. Default arg​
class Calculator:
    # We set z to 0 by default. If the user doesn't provide it, it stays 0.
    def add(self, x, y, z=0):
        return x + y + z

calc = Calculator()

# Mimicking 2-argument overloading
print(calc.add(5, 10))       # Output: 15 (z automatically becomes 0)

# Mimicking 3-argument overloading
print(calc.add(5, 10, 20))   # Output: 35

b. Variable args​
class AdvancedCalculator:
    # *args will capture every single number passed into a tuple named 'args'
    def add(self, *args):
        total = 0
        for number in args:
            total += number
        return total

calc = AdvancedCalculator()

# You can now call the exact same method with 2, 3, or even 5 numbers!
print(calc.add(5, 10))             # Output: 15
print(calc.add(5, 10, 20))         # Output: 35
print(calc.add(1, 2, 3, 4, 5))     # Output: 15

c. Overloading Based on Type​

Sometimes in other languages, you overload a function not because of the number of inputs, but because of their type (e.g., wanting add(int, int) to behave differently than add(str, str)).

In Python, you can achieve this inside a single function using isinstance() to check the data type:

class Greeter:
    def greet(self, name):
        # If they pass a list of names
        if isinstance(name, list):
            for n in name:
                print(f"Hello, {n}!")
        # If they just pass a single string name
        else:
            print(f"Hello, {name}!")

welcome = Greeter()
welcome.greet("Alice")          # Output: Hello, Alice!
welcome.greet(["Bob", "Charlie"]) # Output: Hello, Bob! \n Hello, Charlie!

3. Encapsulation​

  • Building of data (attributes) and methods (functions) within a class, restricting to access some components to control interactions.
  • In Python, we do this using underscores to signal that a variable is "private."
class BankAccount:
  def __init__(self, owner, balance):
      self.owner = owner
      self.__balance = balance  # Double underscore makes it private

  # Getter method to safely look at the balance
  def get_balance(self):
      return self.__balance

  # Setter method to safely change the balance with validation
  def deposit(self, amount):
      if amount > 0:
          self.__balance += amount

account = BankAccount("Alice", 1000)
# print(account.__balance) # ERROR! Python protects this.
account.deposit(500)
print(account.get_balance()) # Output: 1500

4.Data Abstraction​

Abstraction hides the internal implementation details while exposing only the necessary functionality.It focus on "what to do" rather than "how to do".

from abc import ABC, abstractmethod

class Vehicle(ABC): # Abstract Class
    @abstractmethod
    def start_engine(self):
        pass

class Car(Vehicle):
    def start_engine(self):
        return "Car engine started. Vroom!"

# my_vehicle = Vehicle() # ERROR: You cannot create an instance of an abstract class directly.
my_car = Car()
print(my_car.start_engine()) # Output: Car engine started. Vroom!

Advanced OOPs Concepts​

1. The Diamond Problem,MRO and C3 Linearization​

Diamond Problem​

The Diamond Problem occurs when a child class inheritance from two parent classes, And both of those parent classes inherit from the same grandparent class.If both parents override a method from the grandparent, and the child calls that method, which parents method does it use?

[ Grandparent (A) ]  -> has a method: clean_room()
       /          \
  [Parent B]    [Parent C] -> both override clean_room()
       \          /
       [ Child D ]         -> calls clean_room(). Who wins? B or C?

Method Resolution Order(MRO)​

  • To prevent confusion, every class in Python has an MRO a strict , flat list that tells Python exactly which order to search for a method.
  • Python searches from left to right.
  • we can inspect by using Class.__mro__

C3 Linearization (The Math Engine)​

How Python builds that MRO without breaking things? It uses an algorithm called C3 Linearization. C3 enforces three strict rules.

  1. Preserve Subclass Order : Children are always checked before parents.
  2. Preserve Local Precedence : If you write class D(B,C):, B must always checked before C.
  3. Monotonicity : If class B comes before class C in an inheritance chain, it must stay that way across the entire application.

NOTE : If you try to design a messy inheritance tree that breaks these mathematical rules, Python will refuse to compile it and raise a TypeError: Cannot create a consistent method resolution order (MRO).

2. The super() Function​

  • Most People thinks super() just calls the parent class.In sigle inheritance it does.
  • But in Multiple Inheritance, super() actually calls the next class in MRO list, not neccessarily the parent.

class A:
    def process(self):
        print("A's process")

class B(A):
    def process(self):
        print("Entering B")
        super().process() # Calls the next thing in the MRO list
        print("Leaving B")

class C(A):
    def process(self):
        print("Entering C")
        super().process() # Calls the next thing in the MRO list
        print("Leaving C")

class D(B, C):
    def process(self):
        print("Entering D")
        super().process()
        print("Leaving D")

# Let's look at D's MRO: D -> B -> C -> A -> object
d = D()
d.process()

The Output Sequence Notice how C gets executed even though B is not a parent of C!

Entering D
Entering B
Entering C
A's process
Leaving C
Leaving B
Leaving D

3. ABC : Abstract Base Class​

  • The ABC class is strict structural contract.
  • It allows you to define a template class that can't be instantiated (writing internal logic or code) on its own, forcing any child classes to implement (writing the instruction) specific methods.
  • If you want to be a subclass of AppPlugin, you absolutely must write the your own run() method.

from abc import ABC, abstractmethod

class AppPlugin(ABC): # Inheriting from ABC makes it abstract
    
    @abstractmethod
    def run(self):
        """Every child class must implement this method"""
        pass

class MyPlugin(AppPlugin):
    # If we forget to write run(), Python will crash when we create the object
    def run(self):
        return "Plugin is executing!"

plugin = MyPlugin() # Works perfectly!

4.__init_subclass__​

  • Introduced to make class customisation simpler.
  • __init_subclass__ is method inside the parent class.
  • It triggers automatically when a new child classs inherits from it .
  • It helps parent class to inspect,validate, or register child classes the exact millisecond they are defined, before you ever create an instance.
class Registry:
    plugins=[]
    def __init_subclass__(cls,**kwargs):
        super().__init_subclass__(**kwargs)
        #Automatically keep list of all subclasses created
        Registry.plugins.append(cls)
        print(f"Automatically registered plugin: {cls.__name__}")

class AudioPlugin(Registry):
    pass

class VideoPlugin(Registry):
    pass

#Output:
#Automatically registered plugin: AudioPlugin
#Automatically registered plugin: VideoPlugin

5. Metaclasses​

  • class is blueprint of creating an Object, then a Metaclass is a blueprint for creating a Class.
  • In Python, classes themselves are objects. By default, all classes in Python are built by a master metaclass called type.
  • A custom metaclass lets you intercept and completely rewrite how python constructs a class object at runtime.
  • NOTE : You rarely need custom metaclasses. __init_subclass__ handles 95% of use cases cleanly. Use metaclasses only if you are building massive libraries or deep frameworks (like Django or Pydantic) where you must fundamentally alter Python class structures.
  • Metaclass The blueprint that builds classes, giving you complete control over class modification.

# A metaclass inherits from 'type' instead of 'object'
class UpperCaseAttributesMeta(type):
    def __new__(mcs, name, bases, class_dict):
        # We intercept the class creation and force all attributes to uppercase
        uppercase_dict = {}
        for key, value in class_dict.items():
            if not key.startsWith("__"): # Leave special dunder methods alone
                uppercase_dict[key.upper()] = value
            else:
                uppercase_dict[key] = value
                
        # Let 'type' actually construct the class with our modified dictionary
        return super().__new__(mcs, name, bases, uppercase_dict)

# We link our class to the metaclass using the 'metaclass' keyword
class User(metaclass=UpperCaseAttributesMeta):
    role = "admin"
    status = "active"

u = User()
# print(u.role)   # CRASH! 'role' doesn't exist anymore.
print(u.ROLE)     # Output: admin (The metaclass rewrote the attribute name!)
