Memory management

Memory Management​

Python handles memory automatically using mechanism like Reference couting and garbage collection.

1-Reference Counting​

  • This is the primary memory management of the python.
  • Every object keeps a reference counter, which tells how many variables are currently pointing to that object.
  • When a new refrence to that object (variable) is created, then counter increases.
  • When a reference is deleted or out of scope, counter decreases.
  • When counter is zero, then no variable is using that object, so python (Garbage collector) automatically deallocates(frees) that memory.

2- Garbage collection​

It is a process in which python automatically frees memory occupied by the objects that are no longer in use.

Fixing reference cycles​

  • Suppose Object A pointing to Object B and Vice versa, then thier reference count will never drops to zero - causing memory leak.
  • To fix this python uses GC.
  • GC uses a mechanism called Heuristics - When a number of allocation minus number of deallocation exceeds a certain threshold, the GC triggers a collection sweep to break the cycles and free the trapped memory. **

3-Memory allocation​

  • The process of reserving the space in computer's memory so that a program can store its data and variable while it runs.
  • In ptyhon this process is automatically doing by interpreter.
  • There is a big difference in performance depends on how the objects are stored and reused.

Performace optimisation​

Object Pooling (Small Integer and String Caching)​
  • Python does an inernal optimisation in memory called - object interning for small immutable objects (like integers -5 to 256 and short strings).
  • Instead of creating a new object (immutable objects) everytime they will reuse the same object to save memory.
a = 10
b = 10
print(a is b)  # True (They point to the exact same memory address!)

a = 300
b = 300
print(a is b)  # False (Outside the cached range, so two separate objects are made)

  • GIL​

Python's Memory management is not thread thread safe , So only one thread executes byte code at a time.

4- Memory division​

  • Memory(RAM) is divided mainly into 2 parts

    • Stack Memory​
      • This is the place where varible are store when python calls a function for temporary, once the function stops executing the stack memory used by these variables will automatically freed.
      • Allocation happens in continues block of memory.
    def func():
      # These variables are created in stack memory
      a=4
      b=[]
      c=''

    When func() is called a,b,c are stoed in memory.As soon as function ends this memory will release automatically.

    • Heap Memory​

      • Heap Memory is where actual objects and values are stored.
      • When a variable is created , its object will store in heap memory.
      • Stack memory stores only the reference(pointer) to this object.
      • Objects in heaps memory can be share among multiple methods or exist even after the has finished executing.
      • This is where large data structures are stored
      • Garbage collection is responsible for cleaning up unused objects from heap memory.

    • Difference b/w Stack and Heap

      • Heap memory is the storage area where all the objects/values are live and Stack memory is the references to them.
      • Heap is more flexible and Stack follows order.

5- Memory Profiling​

Memory Profiling is the process of analyzing a program's memory consumption while it is running. The goal is to find how much memory is being used and where it is (which line of code) being spent.

  • Memory profiling helps to solve 2 issues

    • Memory Leak - memory no longer alive byt trapped eg: trapped in reference cycle.
    • High Memory Bloat - Code isn't memory leaking but it is incredibly inefficent. eg : loading 4gb csv into a python list instead of streaming it line by line.

  • Tools used forMemory Profiling​

    • tracemalloc

      • Introduced in python 3.4
      • Built in module used to track memory allocation in python.
      • It calls into python's internal memory allocators and record eaxctly which file and which line of code allocated piece of memory.
      • Key Capabilities
        • Traceback - It tells exact which file and line of code that created a specific object.
        • Snapshots - You can take picture of memory state at different times and compare them to see exactly what happens.
        • Statics - It can provide high level summaries of total memory usage grouped by file name.
