Function internal

Here is the breakdown of what happend when you define and execute a function.

1- Stack Frames: How python executes a function​

When we call a function, python creates Frame object(also called Stack Frame) and pushes onto Stack memory in runtime.

A stack frame is a temporary chunk of memory that contains everything the function needs to run :

  • Local variable Array : Fast accessing local variables defined inside the function.
  • Evaluation stack : Where intermediate values are stored during calculations.
  • References to code object : The compiled Byte code.
  • Global pointer : a link to the module's global namespace
  • Back to pointer : a link to the frame that called this function.(eg : if inside function1 , it is calling function2 , then function2's back to pointer will be function1.)

When a function returns its frame will be popped off the stack memory and its local variable are totaly destroyed.

NOTE: - For inspecting the frames use python's sys module.Calling sys._getframe() will gives currently executing frame object.

2. LEGB Scope : How to Python looks up variables​

Python uses LEGB rule to check variable names. Namesapce are checked from the inside to out.The point python finds a match it stops looking.

ScopeDescription
LocalVariable defined in the current function
EnclosingVariable in the local scope of any enclosing/outer function(incase of nested function)
Globalvariable defined at the beginning of the module,or declared global inside the function.
Built-inNames preloaded into python(like print, len,ValueError)

OPtimisation under the hood : LOAD_FAST vs LOAD_GLOBAL​

  • Python doesn't lookup local variable by searching dictionary at runtime.Because python knows the local variable at compile time (explains later), and python store them in fixed array within stack frame at run time.Then interpretr fetches the the value of local variables using brilliant fast instruction called LOAD_FAST.
  • But for Global and Built-in variables will change dynamically (values can be updated while program running, or variable can be added or deleted completely from global space.), so python has look up in dictionaries using LOAD_GLOBAL which is slightly slower.

3. Closures : Free Variables and Cell objects​

  • A closure happens when a nested function references a variable from its enclosing(outer) scope, even that outer function finishes executing.
  def make_multiplier(x):
    def multiplier(y):
        return x * y  # 'x' is a free variable
    return multiplier

    times_three = make_multiplier(3)
    print(times_three(5))  # Outputs 15

  • Normally Outer function exits, the stack frame will be destroyed.So does the inner function still access those variable? CEll objects.

  • The Magic : Cell object​

    • when python compiles make_multiplier ,it see the inner function has used the x , instead storing x in stack frame , Python wraps it in a special internal object called Cell Object.
    • The Cell object lives in the heap independent of the stack frame.
    • Both the outer function and inner function will store a reference to the cell object.
    • Even after the outer function make_multiplier's stack frame vanishes, the inner function keeps refrence to the cell object via __closure__ attribute.
    • closure attribute
      • closure attribute is hidden , built-in property found on every python object.It is a tuple of cell objects, where each cell contains a reference to a variable from outer scope that the function needs to remember.
      • if a function not using any varibales from the outer scope its __closure__ attribute is None.

4. *args and kwargs : Dynamic Parameter Unpacking​

  • *args and kwargs accepts a variable number of arguments.
  • *args (Positional arguments): Python collects all excess positional arguments and pack them into a tuple, and assigns it to a variable name args
  • kwargs (Keyworded arguments): Python collects all excess keyworded arguments , pack them into a dictionary, and assigns a variable name kwargs.
  • Becuase of tuple are immutable and cheaper to allocate than lists, Python striclty uses tuple for *args to keep function calls memory effecient.

5. Inspecting the __code__ object​

  • In python functions are class objects.

  • When we write a function , Python compiles source code into code object (<class 'code'>), which is attached to the function as __code__ attribute.

  • code objects contains the raw byte code and metadata to build stack frame.

  • NOTE : at compile time code object only created , which is blueprint for stack frame, stack frame will be created at runtime.

  • __code__ properties

    def example_func(a, b=10):
      secret = "internal"
      return a + b

    • co_varnames

      • a tuple containing all local variable names (parameters followed by local variables)
      • NOTE : Cell object is diffrent , that is the case of nested function , but this is propery of code object.
      print(example_func.__code__.co_varnames) # (a,b,secret)

    • co_argcount

      • The number of positional arguments(including those with default values) 
        print(example_func.__code__.co_argcount)  # 2

    • co_consts

      • A tuple of all literal constants used in the function. (Python stores these here so it doesn't have to recreate them).

      • "Literal constants" include things like numbers (10, 3.14), strings ("hello"), booleans (True, False), and None.

      • why None : In Python, every function must return something. If a function reaches the end of its block without hitting an explicit return statement, or if it executes a bare return, it automatically returns None. Because None is a literal value that the Python interpreter might need to load and return, the compiler automatically injects it into the co_consts tuple just in case.

            print(example_func.__code__.co_consts)    # (None, 10, 'internal')

    • co_code:

    • The raw, unadulterated bytecode string.

      print(example_func.__code__.co_code)      # b'\x97\x00...' (raw bytes)

    Peeking at the Bytecode with dis​

    • To make sense of co_code, you can use the built-in dis (disassembler) module to turn those raw bytes into human-readable assembly instructions:

      import dis
      dis.dis(example_func)

    output

    2           0 RESUME                   0

    3           2 LOAD_CONST               2 ('internal')
                4 STORE_FAST               2 (secret)

    4           6 LOAD_FAST                0 (a)
                8 LOAD_FAST                1 (b)
                10 BINARY_OP                0 (+)
                14 RETURN_VALUE
